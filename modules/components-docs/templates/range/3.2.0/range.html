Diretiva - gumgaRange [Date|Number]

Obs: Todas as diretivas gumgaMin listadas abaixo, possuem o atributo "label" que é opcional, esse atributo é inserido na input, ele é usado na integração com gumgaErrors para indicar em qual campo se encontra o erro. Se o atributo for omitido, a diretiva usará o atributo name do input.



gumgaRangeDate

O componente GumgaRangeDate serve para validar datas mínimas e máximas para entradas em formulários com campos do tipo date.
O valor do atributo/diretiva é obrigatório e deve ser um objeto contendo duas propriedades, min e max com os valores de suas respectivas datas para execução da validação range.


Demostração RangeDate

Exemplo de código

<form name="myForm" gumga-form>
  <label>Data</label>
  <input type="date" name="rangeDate" ng-model="rangeDate" class="form-control" gumga-error gumga-range-date="{min: '1986-12-29', max: '2015-07-20'}" id="rangedate">
</form>

gumgaRangeNumber

O componente GumgaRangeNumber serve para validar números mínimos e máximos em entradas de formulários com campos do tipo number.


Demostração RangeNumber

O número informado não está entre os valores esperados

Exemplo de código

<form name="myForm" gumga-form>
   <label>Valor</label>
   <input type="number" name="rangeNumber" ng-model="rangeNumber" class="form-control" gumga-range-number="{min: 0, max: 20}">
   <p ng-show="myForm.rangeNumber.$error.rangenumber" class="text-danger">O número informado não está entre os valores esperados</p>
</form>
