Gumga Report - Instalação

Módulo e serviço para criação e gerenciamento de relatórios do stimulsoft.

Atualmente existem duas opções para utilização do gumga-report. Incluir o módulo do mesmo em sua aplicação (framework gumga-reports), ou o serviço disponibilizado.


Pré-requisitos módulo Backend

  • Framework Gumga - ^1.5.0
  • Módulo Report Gumga - ^1.2.19

Instalação do módulo report

1 - Em algum módulo de sua aplicação, adicione a seguinte dependência: 

<dependency>
        <groupId>io.gumga</groupId>
        <artifactId>gumga-reports</artifactId>
        <version>2.0.3/version>
</dependency>

Instalação do Serviço report-service

Se preferir, é possível utilizar um serviço separado do gumga-report.

Clique aqui e baixe ou clone o projeto disponível no github para deploy do serviço.

Se você não sabe como fazê-lo, não se preocupe, faça conforme abaixo:

- Deseja clonar pelo github? Primeiramente execute: git clone https://github.com/GumgaMGA/report-service.git em um diretório desejado.

- O próximo passo será compilar o projeto, e para isto será necessário utilizar o Maven. Após instalado, execute no diretório do projeto o seguinte comando: mvn clean install.

- Pronto, se a seguinte mensagem aparecer, estará tudo pronto para o deploy.



- Agora é só copiar os arquivos compilados .war (caso utilize tomcat) ou .jar (caso não utilize tomcat) dentro do diretório api e fazer o deploy.
