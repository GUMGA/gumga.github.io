Adicionando o Spring Cloud Discovery no projeto com o framework GUMGA

Primeiro passo: Entrar no site https://start.spring.io/ para gerar uma aplicação. Esta aplicação será o servidor de descoberta aonde toda aplicação irá se registrar.

Obs:Depois de adicionar as dependências web e eureka server clicar em generate project.

Segundo passo: Depois de gerar a aplicação, é necessário descompactar e abrir a aplicação na IDE e logo em seguida fazer o build.

Terceiro passo:Adicionar as seguintes propriedades no arquivo de configurações do projeto, que se encontra na pasta src/main/java/resources chamado application.properties:

eureka.client.register-with-eureka=false
eureka.client.fetch-registry=false

Quarto passo:Inserir a anotação @EnableEurekaServer na classe Application do projeto.

Quinto passo:Adicionar a seguinte dependência no módulo boot da aplicação: 

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-eureka</artifactId>
    <version>1.2.7.RELEASE</version>
</dependency>

Sexto passo: Inserir no arquivo no properties de configuração do módulo boot as propriedades abaixo, se não existir um arquivo de configuração no módulo boot será necessário criar um arquivo no caminho src/main/java/resources com o nome bootstrap.properties

spring.application.name=nomeDaAplicacao
eureka.client.serviceUrl.defaultZone=http://ip e porta do servidor de descoberta/eureka

Sétimo passo:Após configurar o arquivo properties, adicionar a anotação @EnableDiscoveryClient para que a aplicação procure o servidor de descoberta para se registrar, essa anotação será colocada na classe Application do módulo boot da aplicação.

Oitavo passo:Executar a aplicação que foi criado no primeiro passo e depois executar o projeto em que foi adicionado a anotação @EnableDiscoveryClient para que o projeto se registre na aplicação de descoberta.
